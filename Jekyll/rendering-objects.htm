Rendering Objects

Simple Objects can also be rendered effectively through the use of YAML Front Matter, Liquid Tag Markup and Liquid Output statements.

In the front matter of the page, a key: value set of objects are declared and entered, as follows:

--- {% highlight liquid %} accensions: - name: White Widow landrace: indica thcas: 25% - name: Super Lemon Haze landrace: sativa thcas: 18% - name: Auto Devil landrace: ruderalis thcas: 21% {% endhighlight %} ---

Note. When constructing a set of key: value pair objects using the Liquid templating language, a JSON-like (YAML) semi-colon separates the name of the object from the names of each key: value pair within the object.

In addition, notice also the use again of a single pre-fixing dash symbol, or - when declaring each element of the object.

Liquid For Loop #2

We will now use a Liquid For Loop to extract and display the accession names and other elements of the accessions object by ...

  1. Programmatically iterating over each key: value pair of the object,
  • And by,
  1. Outputing the variable accession at each iteration of the loop.

As follows: 

<ul>
{% for accession in page.accessions %}
;where the object accessions 
	and the key: value pairs of accession
		are declared in the front matter of the page
Name: <li>{{"{{ accession.name " }}}}</li>,
Landrace: <li>{{"{{ accession.landrace " }}}}</li>,
THCAS: <li>{{"{{ accession.thcas " }}}}</li>
{% endfor %}
</ul>

The Object Rendering

When the page is refreshed, the names, landraces, and level of thcas of the accessions now appear in the following unordered list ...

    {% for accession in page.accessions %}
  • {{accession.name}}, {{accession.landrace}}, {{accession.thcas}}
    • {% endfor %}
