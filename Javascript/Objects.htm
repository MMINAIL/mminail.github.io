Javascript Objects

Welcome! This is the Javascript Objects page of the MMINAIL.

Object-objects

The first thing we must do in this exercise is to delare an Javascript object using the Object-object constructor and then assign the new object to a variable. 

Declare a new object with the Object-object constructor
{% highlight html linenos %}
new Object();
{% endhighlight %}
Assign the new constructor to the variable 'hot-dog' ...
Add: Nothing to the argument of the Object() at this time
{% highlight html linenos %}
var hotDog = new Object();
{% endhighlight %}

Object-constructors

Next, we must declare a new string with the Object-string constructor, a member of the class of Object-constructors available for use in Javascript. 

Declare a new string with the Object-string constructor
{% highlight html linenos %}
new String();
{% endhighlight %}
Assign the new constructor to the variable 'hot-string' ...
Add: Some text to the argument of the String() at this time
{% highlight html linenos %}
var hotString = new String('This is a very hot dog!');
{% endhighlight %}

Object-properties

Now, let's add some properties to our new 'hot-dog' object 

Declare hotDog Object properties using 'dot' notation
{% highlight html linenos %}
hotDog.degree = 'Fahrenheit';
hotDog.temperature = 99;
hotDog.cooked = true;
{% endhighlight %}

Function Objects

Time to add a function that will compile the values of the elements of our hotDog Object. 

Declare a Function Object ...
{% highlight html linenos %}
function() {};
{% endhighlight %}

Assign the new function Object to 'fetchElements'
using 'dot' notation ...
Create a method within the curly braces { ... } 
of the function.
{% highlight html linenos %}
hotDog.fetchElements = function() {
    let Sam = ' Sensing the temperature in ' + 
    hotDog.degree + ', ' + 
    'I notice the hot-dog reads ' + 
    hotDog.temperature + ' degrees. ' + 
    'Therefore, the question of whether or not ' +
    'the hot-dog is cooked is undeniably ' + 
    hotDog.cooked + '!';
    return Sam;
};
{% endhighlight %}
Notice the reserved keywords 'let' and 'return' ... 
They represent the 'alpha' and the 'omega' of the 
method of the function.

Extraction

The final step in the process is figuring out how to render the creation in a browser console window. 

Concatenate the Object-string with the Object-function ...
{% highlight html linenos %}
console.log(hotString + hotDog.fetchElements());
{% endhighlight %}
Note. The console.log method is a built-in
function of Javascript ...
The console.log method prints a log statement 
to the browser console window.

Rendering the Paragraph

Hey! We've made a program!

Let's compile and run the program in the Firefox Web-console. 

Compiling the program for simple copy and paste ...
{% highlight html linenos %}
var hotDog = new Object();
var hotString = new String('This is a very hot dog!');
hotDog.degree = 'Fahrenheit';
hotDog.temperature = 99;
hotDog.cooked = true;
hotDog.fetchElements = function() {
    let Sam = ' Sensing the temperature in ' + 
    hotDog.degree + ', ' + 
    'I notice the hot-dog reads ' + 
    hotDog.temperature + ' degrees. ' + 
    'Therefore, the question of whether or not ' +
    'the hot-dog is cooked is undeniably ' + 
    hotDog.cooked + '!';
    return Sam;
};
console.log(hotString + hotDog.fetchElements());
{% endhighlight %}
Copy and paste the above code block into 
the Firefox Web-console ...
Then, hit the Enter key.

Note. You can clear the Firefox Web-console by clicking on the trash-can icon. Be sure to also purge the browser memory prior to execution by re-setting ( re-loading ) the browser window. 

Finito! ( Interlude )

Custom Constructor Functions

Now that we know how to construct a simple Object-object from the built-in Object constructor ...

Let's build a more complex Custom Constructor Function ...

From which we may choose to unleash a fleet of instantiated Like-objects upon the unsuspecting world.

To create a new Custom-object constructor we must first declare, as above, a Function Object. 

Declare the Function Object ...
{% highlight html linenos %}
function() {};
{% endhighlight %}
Looks familiar, heh?

Next, we assign the Function Object to the name of our new class of Like-objects. In this case, assign the Function Object to the Variable 'machinea' thus creating a Custom-object constructor template. 

Assign the Function Object ...
{% highlight html linenos %}
var Machinea = function() {};
{% endhighlight %}
We are not quite ready to unleash ...
Necesitaremos some properties and methods.

Custom-object Constructors

To fill in the blanks of our Custom-object constructor template ...

We must think of the properties and methods of our new class of Like-objects.

For example, our 'Machinea' can fly, specifically they can 'hover'.

Therefore, we can add the 'hover' method to our Custom-object constructor template, as follows: 

Add the 'hover' method to the Custom-object 
constructor template using 'this-dot' notation ...
{% highlight html linenos %}
var Machinea = function() {
    this.hover = function() {
        let Blue = 'I can hover like a bee ..., ' +
        'and, sting like one, too!';
        return Blue;
    };
};
{% endhighlight %}
We've discovered another method ... to 'Sting'!

Add the 'sting' method to the Custom-object 
constructor template using 'this-dot' notation ...

        return Blue;
    };

    this.sting = function() {
        let Red = ' Does that hurt? ...! ';
        return Red;

Next, Add the 'fetchElements' method to 
the Custom-object constructor template.

Add the 'fetchElements' method to the 
Custom-object constructor template using
'this-dot' notation ...
{% highlight html linenos %}
var Machinea = function(count, size, color,
    tentacles, lazer) {
    this.count = count;
    this.size = size;
    this.color = color;
    this.tentacles = tentacles;
    this.lazer = lazer;

    this.hover = function() {

        return Green;

Rendering the Results of the Internal Methods of the Instance

Can we render the results of the methods inside the template, as well?

Answer. Yes, we can ... with a little transformation, as follows: 

Concatenate the three methods of the 
Custom-object constructor template ...
{% highlight html linenos %}
console.log(this.hover() + this.sting() + 
    this.fetchElements());
{% endhighlight %}
Note. The methods inside the Instance template 
must now be rendered relative to the 
new instance 'reportOne'.

The internal methods are now transformed from the template 'this' to the instance 'reportOne'. 

Rendering the results of the Internal methods ...
{% highlight html linenos %}
console.log(reportOne.hover() + reportOne.sting() +
    reportOne.fetchElements());
{% endhighlight %}

the Elements Report line.
